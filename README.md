# SimpleWeight
### Richard Hawkins

SimpleWeight was developed to address the critical user need for a streamlined, secure, and persistent way to track daily weight changes and monitor progress toward personal health milestones. By providing a user-centered mobile experience that simplifies data logging, the application enables individuals to maintain a digital history of their health journey without the friction of manual bookkeeping. The primary requirements focused on robust data persistence through local storage, secure user authentication to ensure privacy, and a proactive notification system that celebrates user achievements, ultimately fostering a long-term commitment to health goals.

To support these objectives, the application features a cohesive set of screens including a secure login interface, a dynamic weight history dashboard, and a transparent SMS notification setup. The user interface designs keep the user in mind by leveraging familiar Material Design patterns such as the Floating Action Button for primary actions and a RecyclerView for efficient data presentation. A key success of the UI is the implementation of a BottomSheetDialog for data entry, which allows users to add or edit weight records without leaving their current context. This design keeps the experience fluid and intuitive, successfully minimizing cognitive load while maintaining a modern, responsive aesthetic.

The development process followed a modular approach that prioritized security and scalability. By implementing a dedicated DatabaseHelper using SQLite, the data management logic was successfully decoupled from the user interface controllers, ensuring that weight records and target goals are stored reliably across app restarts. This strategy, along with the use of the adapter pattern for list management, demonstrates a commitment to clean code principles that can be easily applied to future mobile projects. This approach ensured that the application remains maintainable as features expand, providing a solid foundation for more complex data-driven mobile architectures.

Functionality was verified through a testing process that examined authentication flows, data integrity, and runtime permission handling. Testing was crucial not only for preventing crashes but for revealing edge cases in goal-checking logic and data isolation between different users. Throughout the development lifecycle, innovation was required to overcome the challenges of user privacy and transparency. This was most evident in the custom SMS permission flow, which utilizes a dedicated interstitial screen to explain the value of notifications to the user before requesting system access. Handling this transition gracefully ensured that the app remains functional and trustworthy, even when users opt to limit specific device permissions.

In the final implementation, the database management component stands out as the most successful demonstration of technical knowledge and experience. Successfully managing relational data in a local SQLite environment required a deep understanding of schema design, unique constraints, and efficient query handling. This robust backend ensures that the user's most valuable asset—their history and progress—is handled with the highest level of reliability. By bringing together complex data persistence with a polished, user-centered interface, SimpleWeight effectively showcases a comprehensive mastery of modern mobile application development.
